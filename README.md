# M5Stack ATOMS3R — アナログ入力 / AUXスイッチ CAN コンバーター

**ATOM S3R + Atomic CAN Base + EXT.I/O 2 で作る、スマートフォン操作対応 CAN バス I/O インターフェース**

---

## 概要

車載 ECU（EMTRON など）やデータロガーと CAN バスで接続し、アナログセンサー値の取得・スイッチ出力制御をスマートフォンのブラウザから行えるデバイスです。追加アプリ不要で Web Bluetooth API を使ってリアルタイム操作・モニタリングができます。

走行中のセンサー確認、スポット溶接コントローラーのシーケンス操作、レースカーのスイッチパネル代替など、車載用途を中心に幅広く使えます。

---

## 特長

**8 チャンネル 12bit ADC 読み取り**  
EXT.I/O 2 経由で 0〜3.3V のアナログ電圧を 12bit 精度で読み取ります。分圧抵抗補正係数・乗数・オフセット・単位をチャンネルごとに設定でき、電圧・電流・温度・圧力など任意の物理量に換算してメーター表示します。

**CAN バス送信（最大 2 フレーム / 20Hz）**  
ADC 値を 2byte リトルエンディアン / ビッグエンディアン の選択式で CAN フレームに格納し、設定した TX ID で送信します。ECU が直接読み取れるフォーマットで出力するため、配線一本で既存システムに統合できます。

**8 チャンネル AUX デジタル出力**  
各ピンを個別に INPUT（ADC）モードまたは AUX（デジタル出力）モードに設定可能。AUX ピンはスマートフォンの Keypad 画面からトグル / パルス操作でき、ON/OFF 値を自由に設定してリレーや FET ドライバを制御します。

**BLE Keypad モード（スマートフォン操作）**  
Web Bluetooth API を使い、専用アプリ不要でブラウザから直接 BLE 接続します。AUX ピンの数に応じて Keypad ボタンが自動生成され、ラベル・モード（トグル/パルス）・ON/OFF 値をブラウザ上で設定・保存できます。

**CAN RX モード（外部 CAN 信号による自動制御）**  
EMTRON ECU など外部デバイスが送信する CAN フレームを受信し、バイト値に応じて AUX ピンを自動的に ON/OFF します。受信 ID・バイト位置・ON/OFF 判定値はブラウザから設定可能。Keypad モードと CAN RX モードは排他選択で切り替えます。

**設定の完全永続化**  
TX ID・ボーレート・ピンモード・スイッチ値・係数・エンディアンなど全設定を NVS（不揮発性ストレージ）に保存し、電源再投入後も自動復元します。ブラウザ側の設定も localStorage に保存されるため、再接続後にすぐ使える状態になります。

## ハードウェア構成

| 部品 | 型番 | 役割 |
|---|---|---|
| マイコン | M5Stack ATOM S3R | メイン制御・BLE |
| CAN インターフェース | M5Stack Atomic CAN Base | CAN バス接続 |
| I/O 拡張 | M5Stack EXT.I/O 2 | 8ch ADC / デジタル出力 |

**配線**  
ATOM S3R と EXT.I/O 2 は I2C（Grove ポート）で接続します。Atomic CAN Base は ATOM S3R の底面に直接スタックします。CAN_H / CAN_L を車両または ECU の CAN バスに接続し、ターミネーション抵抗（120Ω）を必要に応じて追加します。

---

## ソフトウェア構成

```
main.py      MicroPython ファームウェア（ATOM S3R 書き込み）
boot.py      WiFi 無効化・起動設定（main.py と同時に書き込み必須）
index.html   Web アプリ（スマートフォンのブラウザで開く）
```

Web アプリは単一 HTML ファイルで完結しており、HTTPS サーバーまたはローカルサーバーから配信します。ファームウェアは Thonny IDE で ATOM S3R に書き込みます。

## 使い方

### 1. ファームウェア書き込み

Thonny IDE で ATOM S3R に接続し、`boot.py` と `main.py` を書き込みます。シリアルモニターに `BLE ready` が表示されれば起動完了です。

### 2. Web アプリを開く

`index.html` を HTTPS サーバーから配信し、Chrome（PC または Android）でアクセスします。iOS は Web Bluetooth 非対応のため使えません。

### 3. BLE 接続

「CONNECT」ボタンをタップ → デバイス一覧から「M5S3R_AnaCAN」を選択 → 接続完了で ADC メーター画面が表示されます。

### 4. ピンモード設定

設定画面の「PIN MODE」セクションで各ピンをタップして INPUT / AUX を切り替え、「ピンモード SET & 再起動」ボタンを押します。設定はデバイスに保存され、再起動後も保持されます。

### 5. ADC 係数設定

「ADC 表示係数」セクションでチャンネルを選択し、乗数・オフセット・単位・表示レンジ・カラーを設定します。分圧抵抗を使用する場合は R1/R2 を入力すると自動補正されます。

### 6. AUX 操作

AUX モードのピンは、ゲージ画面のボタンまたはキーパッド画面のボタンから操作します。CAN RX モードに切り替えると、外部 CAN フレームで自動制御できます。

## 設定項目一覧

| 項目 | 説明 |
|---|---|
| ADC CAN TX ID | ADC 値を送信する CAN フレームの ID（8ch を 2 フレームに分割） |
| SWITCH CAN TX ID | AUX スイッチ状態を送信する CAN フレームの ID |
| CAN ボーレート | 125k / 250k / 500k / 1Mbps から選択 |
| ADC エンディアン | リトルエンディアン（LE）/ ビッグエンディアン（BE） |
| PIN MODE | チャンネルごとに INPUT（ADC）/ AUX（デジタル出力）を設定 |
| AUX 制御モード | Keypad（BLE 操作）/ CAN RX（外部 CAN 自動制御）を排他選択 |
| Keypad ボタン設定 | ラベル・トグル/パルスモード・ON/OFF 値をピンごとに設定 |
| CAN RX 受信設定 | 受信 ID・各ピンの ON/OFF 判定バイト値を設定 |
| ADC 表示係数 | 乗数・オフセット・単位・レンジ・カラー・分圧抵抗補正 |

## 動作環境

**デバイス側**
- MicroPython v1.23 以降（M5Stack 公式ビルド）
- M5Stack ATOM S3R 専用（Bluetooth LE 内蔵が必要）

**ブラウザ側**
- Google Chrome（PC / Android）
- HTTPS 環境必須（`file://` では Web Bluetooth が動作しません）
- iOS / Safari は Web Bluetooth 非対応のため不可

## 技術的な実装ポイント

**I2C プロトコル対応**  
EXT.I/O 2 の STM32 ファームウェアは Repeated START に非対応のため、ADC 読み取り時は `writeto()` + `readfrom()` の 2 トランザクション方式（STOP 挟み）で実装しています。

**BLE RX バッファ拡張**  
MicroPython の BLE GATT 書き込み特性はデフォルトバッファが 20 バイトで、長いコマンドが切り捨てられます。`gatts_set_buffer(rx_handle, 64)` でバッファを拡張しています。

**BLE キューバイパス送信**  
再起動前の `RESTARTING` / `PIN_MODES=` 通知はメインループのキューが処理される前に `machine.reset()` が走るため、`uart.send()` で直接送信しています。

**移動平均の急変対応**  
4 サンプル移動平均バッファは、前回値との差が 0.16V（200 カウント）を超えた場合にバッファ全体を新値でフラッシュし、電圧除去時の応答遅延を解消しています。
